설명

접근 제한·에러 화면이 뜨는 경우

외부 네트워크 차단·프로젝트 비활성화·접속 실패·에러 페이지 등 접근이 막히는 상황과 그 이유, 대처 방법을 설명

Protect Go는 산업 현장 데이터를 다루는 만큼 접근을 엄격하게 제한한다. 화면이 막히거나 에러가 떠도 대부분은 고장이 아니라 의도된 보안 동작이다. 어떤 경우에 무엇이 뜨고 왜 그런지, 어떻게 대처하는지 정리한다.

외부 네트워크에서 접속하면 막힌다

프로젝트는 기본적으로 사내 내부 네트워크에서만 접근할 수 있다. 외부 IP로 접속하면 접근 권한 오류 페이지가 뜨거나, 프로젝트 접속 시 "보안 정책에 따라 외부 네트워크 접속이 차단되었습니다"라는 안내가 표시된다.

  • 대처: 사내 네트워크(내부 IP)에서 접속하거나 VPN으로 연결한 뒤 다시 시도한다.
  • 브라우저에 ERR_CONNECTION_TIMED_OUT이 뜨는 것도 같은 이유일 수 있다.

프로젝트 접속이 멈추거나 실패한다

[프로젝트 접속]을 눌렀는데 진행률이 멈춰 있고 15초 이상 응답이 없으면 접근 실패 안내가 표시된다. 위의 외부 네트워크 문제이거나, 현장 서버 상태 문제일 수 있다.

  • 대처: 네트워크를 확인하고 다시 시도한다. 문제가 계속되면 고객센터로 문의한다.

프로젝트가 비활성화되어 있다

프로젝트 카드에 "서비스 이용 일시 중지"가 표시되고 접속 버튼에 마우스를 올리면 "현재 비활성화되어 접근할 수 없습니다" 툴팁이 뜬다. 이는 공급사 어드민이 해당 프로젝트를 임시 차단한 상태다.

  • 대처: 사용자가 직접 풀 수 없다. 관리자(어드민)에게 문의한다.

로그인을 여러 번 틀렸다

같은 아이디로 5회 이상 로그인에 실패하면 보안 문자(captcha) 화면이 나타난다. 자동 공격을 막기 위한 동작이다.

  • 대처: 화면에 보이는 보안 문자를 정확히 입력한다. 비밀번호가 기억나지 않으면 비밀번호 찾기로 재설정한다.

소속된 프로젝트가 없다

가입은 했지만 아직 어떤 프로젝트에도 속하지 않으면 프로젝트 초대 안내 페이지로 이동한다. 에러가 아니라, 관리자의 초대를 기다리는 정상 상태다.

  • 대처: 프로젝트 관리자에게 초대를 요청한다. 관리자가 보낸 초대 메일의 수락 버튼으로 프로젝트에 합류할 수 있다.

그 밖의 에러 페이지

위에 해당하지 않는 오류는 에러 코드와 함께 에러 페이지가 표시된다. 코드를 확인할 수 없으면 "404"로 표시된다. [홈으로 돌아가기]를 누르면 로그인 상태에서는 대시보드로, 비로그인 상태에서는 로그인 페이지로 이동한다.
