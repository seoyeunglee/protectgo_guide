# Protect Go AI 사용자 가이드 — Problem One-Pager

## 문제 정의
- **메인 문제**: 가이드/설명서를 만드는 사람이 화면·정책이 바뀔 때마다 문서를 수작업으로 갱신해야 해서 최신성이 떨어지고(드리프트), 작성 비용이 크다.
  - 분류 이유: 계획서 v0.7의 핵심 동인이 "가이드를 만드는 작업 = KB 콘텐츠를 구조화하는 작업"이며, 설계 무게중심이 "여러 소스를 필드별 권위로 분리해 KB 엔트리를 조합하고, 변경분만 증분 갱신"하는 데 있다.
- **부차 문제**: Protect Go 사용자가 기능 사용 중 막히면 참고할 최신 가이드가 없어 시행착오를 겪거나 직접 문의해야 한다.
  - 메인 문제가 해결되면 자연히 따라오는 결과이자 가이드의 최종 수혜 지점.
- **범위 외**: 여러 소스에 흩어진 정보를 매번 새로 찾아 조합해야 해서 신규 합류자·타 팀이 제품 동작을 파악하는 데 시간이 오래 걸린다.
  - 조직 전체 지식 검색 비용 절감은 더 큰 비전이나, 2주 범위(가이드+KB 묶음, 1~2 도메인)와 자동 업데이트 에이전트(스텁)의 후속 영역.

## 핵심 가치 가설 (KPI형)
**기존 상세 설명서 작성 방식 대비 가이드 제작·갱신 시간을 단축한다.**

KB 엔트리를 "여러 소스의 조합 + 필드별 출처(provenance)"로 구조화하면, 화면·정책이 바뀌었을 때 영향받는 필드만 다시 만들면 된다. 갱신 단위를 줄여 제작·갱신 시간 자체를 단축하는 구조적 가설이며, 계획서의 지표 설계 방향 "(B) 가이드 관점: 주 대상은 만드는 사람, 작성 시간 단축 수치화"와 정렬한다.
