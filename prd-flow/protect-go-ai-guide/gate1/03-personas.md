# 03. 페르소나

| 페르소나 | 분류 | Job-to-be-Done |
|---|---|---|
| KB/가이드 구축 담당자 (이서영 등) | 메인 | 화면·정책이 바뀔 때 최소 노력으로 가이드를 최신 상태로 유지하고 싶다 |
| Protect Go 일반 사용자 (현장 작업자·안전 관리자 등) | 서브 | 기능 사용 중 막히는 지점에서 바로 답을 찾고 싶다 |
| 어드민·매니저 (운영·관리 권한 보유자) | 서브 | 자기 권한 범위에 맞는 설정·정책 정보를 빠르게 확인하고 싶다 |
| (향후) 자동 업데이트 에이전트 | 범위 외 | 변경된 소스를 감지해 같은 스키마로 KB를 갱신하고 싶다 — 이번엔 스키마·흐름 스텁까지만 |

## 분류 근거
- **메인 = KB/가이드 구축 담당자**: 핵심 가치 가설(KPI형 — 제작·갱신 시간 단축)의 직접 수혜자이자 측정 대상. 이번 작업에서 실제로 KB 엔트리를 만들고 갱신해보는 주체.
- **서브 = Protect Go 일반 사용자, 어드민·매니저**: 가이드의 최종 소비자. Diátaxis 4유형(튜토리얼·하우투·레퍼런스·설명) 분류와 도메인(프로젝트 생성·탐지 노드) 선정의 근거가 되는 대상. 이번엔 정량 측정 대상은 아니지만 콘텐츠 구조 설계 시 고려한다.
- **범위 외 = 자동 업데이트 에이전트**: "에이전트는 스텁(스키마+흐름 설계만, 구현 금지)"이라는 2주 범위 규칙에 따라, 이번 PRD에서 정식 페르소나로 다루지 않고 `agent/flow.md`·`agent/interfaces.md`의 설계 대상으로만 취급한다.
