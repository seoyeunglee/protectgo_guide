# 02. 핵심 가치 가설

## 채택안 (KPI형)
기존 상세 설명서 작성 방식 대비 가이드 제작·갱신 시간을 단축한다.

## 본문
KB 엔트리를 "여러 소스의 조합 + 필드별 출처(provenance)"로 구조화하면, 화면·정책이 바뀌었을 때 영향받는 필드만 다시 만들면 된다. 이는 문서를 처음부터 다시 쓰거나 전체를 검토해야 했던 기존 방식과 달리, **갱신 단위를 줄여 제작·갱신 시간 자체를 단축**하는 구조적 가설이다.

- 메인 문제(드리프트·수작업 갱신 비용)와 직접 정렬: 갱신 시간 단축은 드리프트 감소의 결과이자 측정 가능한 대리 지표.
- 계획서의 지표 설계 방향 (B) "가이드 관점: 주 대상은 가이드 *만드는 사람*이며, 기존 상세 설명서 작성 시간 대비 단축 효과를 수치화"와 정렬.
- 측정 시점: 2주 범위에서는 선정 도메인(프로젝트 생성 + 탐지 노드)의 엔트리 1~2개를 실제로 만들어보며 기존 방식 대비 소요 시간·갱신 범위를 비교한다 (정량 지표는 Auto Phase에서 구체화).
