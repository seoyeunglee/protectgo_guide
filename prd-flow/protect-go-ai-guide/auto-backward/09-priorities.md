# 09. 우선순위 (P0 / P1 / P2)

> 분류 기준: 메인/서브 페르소나 영향 범위 + 핵심 가치 가설(KPI) 영향도
> 입력: `gate1/03-personas.md`, `auto-backward/07-features.md`

## P0 — 가설 검증의 핵심, 메인 페르소나 직접 영향

### F1 · KB 엔트리 스키마 적용 (선정 도메인)
- 메인 페르소나(KB/가이드 구축 담당자)의 작업 구조 자체를 바꾸는 토대
- F2·F6의 선행 조건 — 이 항목 없이는 가설 검증도, 가이드 출력도 불가능

### F2 · 실제 엔트리 제작 및 비교 검증
- 핵심 가치 가설("제작·갱신 시간 단축")을 직접 검증하는 유일한 항목
- Devil's Advocate moderate 반론에 대한 명시적 대응 — 누락 시 V1 전체가 "추정에 머무는" 상태로 끝남

### F6 · 가이드 웹페이지 출력
- 일반 사용자, 어드민·매니저 서브 페르소나가 실제로 가치를 얻는 접점
- 부차 문제("최신 가이드 부재로 인한 시행착오")가 해소되는지 확인 가능한 유일한 산출물

## P1 — 핵심을 보강, 효과를 키우는 항목

### F3 · Diátaxis × 도메인 콘텐츠 템플릿 및 작성 체크리스트
- F2의 제작 속도와 일관성에 직접 기여하나, 템플릿이 없어도 F2 자체는 진행 가능
- 노드·조건 분기 화면용 갈래 분리는 P1 안에서 우선 검토

### F5 · 소스 변경 영향 수동 점검 체크리스트
- 메인 페르소나의 "최소 노력으로 최신 상태 유지" JTBD에 직접 대응
- F1·F2가 끝난 뒤에도 의미가 유지되는 운영 단계 항목 — 동시 착수보다 후행 가능

## P2 — 후반부, 순서 제약이 있는 보조 항목

### F4 · 권위 소스 안내 (신규 문서 생성 없음)
- 효과는 명확하나 구현 난이도가 낮고, 기존 표(`schema/kb-entry.md`)에 안내 문구만 추가하면 되는 경량 작업
- F1 적용 과정에서 자연스럽게 함께 정리 가능

### F7 · kb/ to guide/ 렌더링 빌드 스크립트
- Devil's Advocate moderate 반론(가설과의 연결이 약함, 순서 역전 위험)에 따라 P0 항목(F1·F2·F6) 완료 이후 착수
- V1 종료 시점까지 "수동 빌드 확인" 수준으로도 F6 출력은 가능하므로, 자동화 자체는 후순위

### F8 · 라이브 배포 소스 참조 + 동기화 트리거
- F7과 같은 이유로 P2 — 토대(F1)와 권위 소스 안내(F4)가 갖춰진 뒤에 의미가 있는 보조 점검·동기화 도구
- draft(임시) 출처를 다루므로 확정 출처 우선순위(F1·F4)가 먼저 자리잡아야 권위 등급 충돌 없이 동작
- V1 종료 시점까지 "변경 감지 + 메타데이터 동기화" 수준까지만, 자동 승격·자동 트리거는 범위 외

## 순서 제약 요약

F1과 F2가 선행되고, F6(가이드 출력)이 그 결과를 사용자에게 보여주는 통로로 뒤따른다. F3·F5는 핵심 작업 흐름을 보강하는 병행 항목이다. F4는 F1 적용 과정에 자연스럽게 포함된다. F7과 F8은 토대(F1·F2)와 출력 형태(F6)·권위 소스 안내(F4)가 갖춰진 뒤 가장 마지막에 착수한다.
