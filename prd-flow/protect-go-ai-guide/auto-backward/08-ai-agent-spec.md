# 08. AI 에이전트 사양 (스텁)

> 범위: 계획서 v0.7 "에이전트는 스텁" 규칙에 따라 **인터페이스 계약과 흐름 설계까지만** 다룬다. 실제 동작 구현은 본 V1의 범위 밖이다.
> 근거 문서: `agent/flow.md`(흐름 개요), `agent/interfaces.md`(타입·함수 시그니처)

## 위치와 역할

자동 업데이트 에이전트는 메인 페르소나(KB/가이드 구축 담당자)의 JTBD 중 "최소 노력으로 최신 상태 유지"를 장기적으로 자동화하는 후속 단계다. `gate1/03-personas.md`에서 "범위 외(스키마·흐름 스텁까지만)"로 명시 분류됐고, V1에서는 그 경계를 그대로 지킨다.

## 흐름 개요 (4단계, 호출 순서 고정)

판별에서 시작해 검색·매핑, 구조화·생성을 거쳐 반영으로 끝나는 단방향 흐름이다. 각 단계는 바로 앞 단계의 산출만 입력으로 받으며 역방향 의존이 없다.

1. **판별**: 변경된 소스(PRD, 화면설계서 PDF, 프론트엔드 코드, 프로토타입, Figma, 티켓)를 받아 변경 유형(신규/수정)과 영향받는 필드·카테고리 후보를 추정
2. **검색·매핑**: 판별 결과를 기존 KB 엔트리 목록과 대조해 대상 엔트리(또는 신규 여부)와 영향받는 필드를 좁힘
3. **구조화·생성**: 필드별 권위 소스 원문을 받아 엔트리 전체(신규) 또는 변경 필드만(수정)과 출처 정보를 구성
4. **반영**: 구성 결과를 KB 경로에 기록하고 가이드 빌드 트리거 여부를 반환

## 공유 타입 — KB 엔트리

`schema/kb-entry.md` 프런트매터와 1대1 대응한다. 식별자, 카테고리, Diátaxis 유형, 제목·설명·정책·화면 참조·인수 조건·시각 자료, 필드별 출처 정보(출처 종류·참조 위치·버전), 언어, 갱신 시각으로 구성된다. 출처 정보의 버전 값이 증분 갱신 판단의 단일 기준이다.

## V1과의 접점 — 설계 입력으로서의 연결고리

에이전트를 구현하지는 않지만, V1의 F5(소스 변경 영향 수동 점검 체크리스트)는 흐름의 1·2단계(판별, 검색·매핑)가 사람의 손으로 어떻게 수행되는지를 보여주는 선행 사례가 된다. AI Agent Engineer 페르소나가 합의 메모에서 강조한 지점으로, 향후 자동화 단계를 설계할 때 V1의 체크리스트 항목과 점검 흐름을 1차 입력으로 참고할 수 있다.

## 비범위 (V1에서 다루지 않음)

- 각 단계의 실제 구현(추출 로직, 매핑 알고리즘, 생성 프롬프트, 저장소 연동)
- 오류·재시도·동시 실행 처리
- 트리거 연결(웹훅, 정기 실행 등)
- 위 항목은 모두 후속 범위이며, V1에서 착수 시 Hard Stop H1(기존 사양 충돌) 대상이 된다.
