# Discovery 자동 진행 리포트

## 컨텍스트
- 모드: pg (Protect GO AI)
- 도메인 지식 검증: protect-go-knowledge 적용
- 산출물 위치: `prd-flow/protect-go-ai-guide/`

## 거시 의사결정 (Gate 1·1.5에서 확정)

- **메인 문제**: 가이드/KB 작성자가 화면·정책이 바뀔 때마다 문서를 수작업으로 갱신해야 해서 최신성이 떨어지고(드리프트), 작성 비용이 큼
- **핵심 가치 가설**: 기존 상세 설명서 작성 방식 대비 가이드 제작·갱신 시간을 단축한다 (KB 엔트리를 소스 조합 + 필드별 provenance로 구조화 → 변경 영향 필드만 재작성)
- **메인 페르소나**: KB/가이드 구축 담당자 — "최소 노력으로 가이드를 최신 상태로 유지"
- **채택 솔루션**: S1(스키마 적용 + 실제 엔트리 제작·검증)을 토대로 S2(템플릿)·S4(권위 소스 안내)·S5(점검 체크리스트)를 보조로 결합, S6(렌더링 빌드)은 후반부로 순서 고정
- **V1 범위**: F1(스키마 적용) → F2(엔트리 제작·검증) → F6(가이드 출력)을 핵심으로, F3·F5(보강)·F4(경량 보조)를 병행, F7(빌드 스크립트)은 마지막

## 자동으로 내려진 주요 결정 7개

1. 솔루션 발산 시 "자동 영향 탐지·재생성 도구"를 매트릭스에서 제외 (페르소나 분류·CLAUDE.md 스텁 규칙과 직접 충돌 — Hard Stop H1 대상이므로 발산 단계에서 필터링)
2. S4(권위 매핑)를 별도 문서가 아닌 `schema/kb-entry.md` 기존 표 활용 방식으로 좁힘 (Devil's Advocate 반론 — 매핑 문서 자체가 드리프트 대상이 될 수 있음)
3. S5 체크리스트를 "소스 1개당 1~2문항"으로 길이 제한 (Devil's Advocate 반론 — 길어지면 확인 비용이 가설과 반대로 작용)
4. F6(가이드 웹페이지 출력)을 F1·F2의 "출력 통로"로 명시해 기능 목록에 포함 (Gate 1.5 산출물에는 직접 명시되지 않았으나, S1 채택의 자연스러운 귀결로 도출)
5. F7(렌더링 빌드 스크립트)을 P0·P1이 아닌 P2로, 그리고 V1 내 마지막 순서로 고정 (Devil's Advocate·Staff Engineer의 순서 역전 위험 반론 반영)
6. KPI를 "엔트리 제작 시간"과 "갱신 단위 크기" 2개 지표로 구체화 (가설의 "시간 단축"을 측정 가능한 형태로 분해)
7. QA 시나리오에 "자동 갱신이 동작하지 않음을 확인하는" 경계 시나리오를 포함 (스텁 단계의 정상 동작 범위를 명시적으로 검증 대상에 포함)

## Silent Defaults (2건 — 임계값 3 미만)

- **SD-001**: 비교 검증 대상 엔트리를 "정책 복잡도가 높은 화면(탐지 시나리오·노드 1개 + 프로젝트 생성 1개)"으로 추정 선정. 검토된 대안은 최단순 화면 우선, 무작위 선정
- **SD-002**: KPI 비교 기준선을 "GS 인증 사용설명서 작성 경험 또는 작성자 인터뷰 기반 추정치"로 임시 설정. 검토된 대안은 정성 비교만 수행, 기준선 측정 자체를 후속으로 이연

상세 근거는 `auto-backward/12-decision-log.md` 참조.

## Devil's Advocate 코멘트 (요약)

- **moderate 2건** (S1, S6) — 둘 다 Gate 1.5에서 V1 범위·순서로 명시 반영 완료 (S1은 F2 검증 액션으로, S6은 F7의 후반부 순서 고정으로)
- **minor 4건** (S2, S3, S4, S5) — 모두 기능 정의(F3~F5) 작성 시 권고 내용을 그대로 반영 (템플릿 갈래 분리, 도구화 범위 한정, 표 단일화, 체크리스트 길이 제한)
- **critical 0건** — Hard Stop H4 발동 없음

상세는 `auto-forward/05-devil-advocate.md` 참조.

## 페르소나 합의 점수

- 강한 합의 4건 (S1·S2·S4·S5), 약한 합의 2건 (S3·S6), 의견 갈림 0건 — Hard Stop H2 미발동
- AI Agent Engineer는 제외된 "자동 영향 탐지" 방향에 동의하면서, F5(점검 체크리스트)가 향후 에이전트 ①②단계 설계의 입력이 될 수 있다는 연결고리를 제시 (`auto-backward/08-ai-agent-spec.md`에 반영)

## Hard Stop 발동 이력

없음 (H1~H6 전부 미발동)
