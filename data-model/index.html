홈 / 데이터·모델 관리

데이터·모델 관리

탐지가 동작하려면 재료가 먼저 준비돼야 한다. 어떤 센서·카메라·DB에서 데이터를 받을지, 어떤 문서를 참고할지, 어떤 AI 모델을 쓸지를 이 도메인에서 관리한다.

세 가지 재료

메뉴(GNB) 무엇을 관리하나
연결 데이터 관리 센서·카메라·외부 DB(InfluxDB) 같은 데이터 소스와 엣지 디바이스
문서 관리 안전·설비 매뉴얼 PDF (벡터 DB에 적재)
모델 관리 AI 모델의 버전과 배포 대상

연결 데이터 — 탐지의 입력원

영상(RTSP)·센서·열화상·데이터베이스 4가지 유형으로 데이터 소스를 등록한다. 등록된 소스는 탐지 시나리오의 데이터 노드에서 불러 쓴다. 연결 상태는 1분마다 점검되며, 오래 끊기면 자동으로 탐지가 비활성화되는 등 운영 안정성 정책이 함께 동작한다.

모델 — 무엇으로 판정하는가

AI 모델은 버전으로 관리되고, 설비·엣지 디바이스 단위로 배포된다. 같은 탐지라도 어디에 배포된 모델을 쓰는지가 결과를 좌우하므로, 우선순위(엣지 > 설비 > 프로젝트)가 핵심 개념이다. 모델·데이터·설정을 묶은 것이 AI 패키지다.

튜토리얼

처음부터 끝까지 따라 하며 익히는 단계별 안내. 전체 탐지 흐름은 [탐지 시나리오·노드](../../detection-node/tutorial/) 도메인을 참고한다. (준비 중)

하우투

  • 모델 등록·배포하기 — 모델 관리에서 모델의 기본 정보·버전 이력을 확인하고, 기본 적용 버전과 설비·디바이스별 배포 대상을 관리하는 방법
  • 문서 등록·관리하기 (벡터 DB) — 문서 관리에서 폴더를 만들고 PDF 매뉴얼을 업로드해 벡터 DB에 적재하고, 등록 상태를 확인하는 방법
  • 연결 데이터 등록·관리하기 (영상·센서·열화상·InfluxDB) — 연결 데이터 관리에서 영상(RTSP)·센서·열화상·데이터베이스(InfluxDB) 소스를 등록하고 연결 상태를 관리하는 방법

레퍼런스

  • 연결 상태와 자동 비활성화 정책 — 데이터 연결 상태 4종의 정의와 갱신 주기, 연결 불량 안내·자동 비활성화(deactive) 기준을 정리한 레퍼런스
