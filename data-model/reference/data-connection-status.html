레퍼런스

연결 상태와 자동 비활성화 정책

데이터 연결 상태 4종의 정의와 갱신 주기, 연결 불량 안내·자동 비활성화(deactive) 기준을 정리한 레퍼런스

등록한 데이터 소스가 실제로 데이터를 받고 있는지는 연결 상태로 관리된다. 상태가 무엇을 뜻하고, 끊기면 어떻게 동작하는지 정리한다. 등록 절차는 연결 데이터 등록·관리하기를 참고한다.

연결 상태 4종

상태 정의
정상 연결 데이터가 정상 수집됨
연결 진행 중 등록 직후 첫 데이터 적재 전 (외부 DB는 메타데이터 저장 완료 전)
연결 실패 연결 진행 중 상태가 5분 경과
연결 없음 데이터가 수집되지 않음
  • 상태는 1분마다 갱신된다.
  • 측정 항목이 여러 개인 센서는 하나라도 미수집이면 "연결 없음"으로 본다.

끊기면 — 단계별 동작

경과 동작
1시간 연결 없음 에디터에게 데이터 연결 불량 안내 팝업 발송(소스명·유형·장소 표시)
3시간 연결 없음 해당 센서가 연결된 탐지 설정·AI 패키지 노드가 자동 비활성화(deactive)
정상 연결 복귀 노드가 자동으로 active로 복귀

비정상 수집으로 판단하는 경우: 특정 측정 항목이 1시간 동안 소수점 다섯째 자리까지 같은 값이거나, 1시간 동안 1개 이상 측정 항목이 수집되지 않을 때.

삭제 시

데이터 소스를 삭제하면 그 소스가 연결된 탐지 시나리오가 함께 삭제된다. (현재는 시나리오·장소 종속성에 따른 임시 정책으로, 추후 변경 예정.)
