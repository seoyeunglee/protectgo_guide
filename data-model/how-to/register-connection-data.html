연결 데이터 등록·관리하기 (영상·센서·열화상·InfluxDB)

연결 데이터 관리에서 영상(RTSP)·센서·열화상·데이터베이스(InfluxDB) 소스를 등록하고 연결 상태를 관리하는 방법

탐지의 입력이 될 센서·카메라·DB를 등록하는 곳이다. 연결 데이터 관리(GNB 4번째)에서 하며, 등록·삭제는 에디터 권한이 필요하다.

데이터 소스 등록하기 — 4가지 유형

[데이터 소스 등록]을 누르면 유형을 고른다. 어느 유형이든 데이터 소스명(2~20자, 중복 불가)·설치 장소·설비 정보(선택)는 공통이다.

유형 추가 입력 연결 확인
영상 데이터 RTSP URL (rtsp://로 시작) 호출 후 15초 무응답 시 실패
센서 데이터 센서 유형 → 센서명 선택 측정 항목 리스트 표시
열화상 카메라 RTSP URL 셀렉박스 최고/최저/평균 온도 툴팁
데이터 베이스 InfluxDB 접속 정보 (아래 참고)

RTSP URL이 rtsp://로 시작하지 않으면 "올바르지 않은 rtsp url입니다" 에러가 뜬다. 외부 네트워크에서는 영상이 표시되지 않으니 사내망에서 확인한다.

InfluxDB 연결하기

데이터 베이스 유형은 현재 InfluxDB만 선택할 수 있다.

  1. 호스트·포트·사용자 정보를 입력하면 [비밀번호 입력] 버튼이 활성화된다.
  2. 비밀번호(별도 팝업)와 토큰(최대 200자)을 입력한다.
  3. [연결 확인]을 누른다. 성공하면 "연결에 성공했습니다", 실패하면 사유별 안내(접속 정보·권한·서버 상태)가 뜬다.
  4. 확인 후 저장한다. 접속 정보를 다시 바꾸면 확인 상태가 초기화되니 [연결 확인]을 다시 한다.

연결 상태 관리하기

등록한 소스는 리스트에서 연결 상태로 관리된다.

상태 의미
정상 연결 데이터가 정상 수집됨
연결 진행 중 등록 직후 첫 데이터 적재 전
연결 실패 진행 중 상태가 5분 경과
연결 없음 데이터가 수집되지 않음

상태는 1분마다 갱신된다. 연결 없음이 3시간 지속되면, 그 센서가 연결된 탐지 설정·AI 패키지 노드가 자동으로 비활성화(deactive)되고, 다시 정상 연결되면 자동으로 활성(active)으로 돌아온다. 1시간 이상 끊기면 에디터에게 안내 팝업이 발송된다.

삭제 시 주의

데이터 소스를 삭제하면 그 소스가 연결된 탐지 시나리오가 함께 삭제된다. "삭제시 관련된 탐지 시나리오가 모두 삭제 됩니다" 경고를 확인하고 진행한다.
