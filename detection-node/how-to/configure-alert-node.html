알림·제어 노드 설정하기 (대시보드·모바일·담당자 자동 배정)

이벤트 노드 뒤에 대시보드 팝업·모바일 푸시·담당자 자동 배정 알림 노드를 붙여 이상 발생 시 누구에게 어떻게 알릴지 정하는 방법

알림·제어 노드는 흐름의 마지막이다. 앞 단계에서 정한 이벤트가 발생했을 때 누구에게, 어떤 경로로 알릴지를 정한다. 이벤트 설정 노드 뒤에 붙인다.

한 이벤트 설정에는 대시보드 알림 1개 + 모바일 알림 1개까지 붙일 수 있다.

대시보드 팝업 알림 설정하기

  1. 이벤트 노드에 대시보드 알림 노드를 연결한다.
  2. 이상 발생 팝업 알림 스위치를 ON으로 한다.
  3. 이제 이벤트가 발생하면 대시보드에 팝업이 뜬다. 팝업은 2시간 후 자동으로 닫히고, 같은 이벤트에 대해서는 하나만 표시된다.

모바일 푸시 알림 설정하기

  1. 이벤트 노드에 모바일 알림 노드를 연결한다.
  2. 대상 모드를 고른다. - 전체 알림: 프로젝트에 등록되고 앱을 설치한 모든 유저에게 발송된다. 선택 시 전원 발송을 알리는 확인 팝업이 뜬다. - 담당자 지정: 지정한 유저 중 앱 설치 유저에게만 발송된다.
  3. 담당자 지정이라면 [대상 추가]로 유저를 검색해 더한다(이름·소속·전화번호로 검색).
  4. 선택한 유저 중 이 장소의 담당자가 아닌 사람이 있으면 확인 팝업이 뜬다. 모바일 알림은 그 장소 담당자에게만 가므로, [모두 담당자로 지정하고 알림 발송] 또는 [기존 담당자에게만 알림 발송] 중 고른다.

모바일 알림은 앱 설치 토큰이 있는 유저에게만 발송된다. 앱 미설치 유저에게는 가지 않으며, 토큰이 없어 실패한 이벤트는 나중에 앱을 설치해도 푸시·이상 탐지 알림 목록에 다시 나타나지 않는다.

담당자 자동 배정(SOP) 알림 설정하기

  1. 담당자 자동 배정 알림 노드를 연결한다. 처음에는 전체 비활성화 상태다.
  2. 스위치를 활성화하면, 이벤트 발생 시 장소·설비·탐지 유형별로 미리 지정된 담당자에게 자동으로 푸시가 발송된다.

다음 단계

  • 알림을 받은 뒤 현장에서 처리하는 흐름은 이상 상황 처리하기를 참고한다.
  • 발생 조건 자체를 바꾸려면 이벤트 설정 노드 구성하기로 돌아간다.
