홈 / 탐지 시나리오·노드

탐지 시나리오·노드

Protect Go의 탐지는 코드나 설정 파일이 아니라 노드를 연결한 그림으로 정의한다. 탐지 시나리오는 "어떤 데이터를, 어떤 기준으로 감시하고, 이상이 생기면 누구에게 알릴지"를 하나의 흐름으로 묶은 단위다 — 시나리오 하나가 캔버스 한 장이고, 캔버스 위의 노드들이 그 흐름을 구성한다.

데이터가 알림이 되기까지

노드는 항상 왼쪽에서 오른쪽으로 흐른다. 입력에서 시작해 판정과 조건을 거쳐 출력으로 끝난다.

  1. 데이터 노드가 센서·카메라의 측정값과 영상을 받아들인다.
  2. 탐지 설정 노드(또는 AI 패키지 노드)가 그 데이터를 기준과 비교해 이상 여부를 판정한다.
  3. 이벤트 설정 노드가 "기준값을 몇 번 넘으면 이벤트로 볼 것인지" 발생 조건을 정의한다.
  4. 알림 및 제어 노드가 발생한 이벤트를 대시보드 푸시, 모바일 푸시, SOP 등으로 전달한다.

영상 탐지에는 관심 영역을 지정하는 ROI 노드가 추가로 쓰인다.

탐지 시나리오 5종

감시하려는 데이터에 따라 시나리오 유형이 정해진다.

유형 데이터 소스 판정 방식
영상 이상 탐지 CCTV·열화상 영상 비전 AI 모델 (작업자 안전 모델 등)
DB 임계치 탐지 외부 DB 수치 임계치 비교
센서 임계치 탐지 진동·온도·복합가스·열화상 센서 수치 임계치 비교
통계 분석 탐지 센서(다중) Z-Score / 변화율 / EWMA 통계 이상치
발전 효율 저하 탐지 전력 센서 + 발전량 데이터 발전량 예측값 대비 실측값 비교

왜 노드 방식인가

  • 흐름이 보인다: 데이터부터 알림까지 한 화면에서 추적할 수 있어, 알림이 왜 왔는지 연결을 따라가며 확인할 수 있다.
  • 조합이 자유롭다: 같은 데이터 노드를 여러 탐지 설정에 연결하거나, 여러 데이터를 하나의 이벤트로 묶는(융합 이벤트) 구성을 그림으로 만들 수 있다.
  • 검증이 내장된다: 연결할 수 없는 노드 조합은 연결 시점에 차단되고 안내 팝업이 표시된다.

튜토리얼

  • 첫 탐지 시나리오 만들기 — 탐지 설정에서 시나리오를 생성하고, 캔버스에서 노드를 연결해 탐지를 시작하는 과정

하우투

  • AI 패키지 노드 사용하기 — 모델·데이터·설정을 묶은 AI 패키지 노드를 설비 또는 장소 단위로 적용해 탐지에 쓰는 방법
  • 데이터 노드 연결하기 (센서·CCTV·열화상·DB) — 시나리오 캔버스에 데이터 노드를 놓고 설치 장소의 데이터 소스를 불러와 연결하는 방법과, 영상 데이터의 탐지 영역(ROI) 설정
  • 알림·제어 노드 설정하기 (대시보드·모바일·담당자 자동 배정) — 이벤트 노드 뒤에 대시보드 팝업·모바일 푸시·담당자 자동 배정 알림 노드를 붙여 이상 발생 시 누구에게 어떻게 알릴지 정하는 방법
  • 엣지 디바이스 노드 배치하기 — 현장에 배포된 엣지 디바이스를 노드로 불러와, 디바이스에 연결된 센서와 탐지 정보를 확인하고 이벤트 노드를 연결하는 방법
  • 이벤트 설정 노드 구성하기 — 탐지 시나리오 캔버스에서 이벤트 설정 노드를 탐지/AI 패키지/엣지 디바이스 노드와 연결하고, 발생 조건(기준 설정값·발생 건수)을 설정해 저장하는 방법
  • 이상 상황 알림 확인하기 — 헤더의 알림 센터에서 이상 상황 알림을 확인·정리하고, 실시간 이상 발생 알림에서 상세 화면으로 이동하는 방법
  • 탐지 설정 노드 구성하기 — 데이터 노드에 탐지 설정 노드를 연결해 영상 탐지 대상이나 센서 임계치를 정의하는 방법과, 연결 가능한 데이터 제약

레퍼런스

  • 노드 연결 규칙·단축키 — 노드 간 연결 제약, 시나리오 캔버스 단축키, 저장 실패·통신 오류 시 동작을 정리한 레퍼런스
  • 노드 카탈로그 레퍼런스 — 시나리오 캔버스에서 사용할 수 있는 노드 목록과 노드 공통 사양(이름 규칙·삭제 동작)

설명

  • 탐지는 언제 시작되고 어떤 모델을 쓰는가 — 노드를 놓는 것만으로는 탐지가 동작하지 않는 이유(이벤트 노드 연결 조건)와, 같은 탐지에서 어떤 모델이 우선 적용되는지(엣지>설비>프로젝트)를 설명
