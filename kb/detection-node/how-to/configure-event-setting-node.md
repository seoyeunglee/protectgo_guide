# 이벤트 설정 노드 구성하기

탐지 시나리오 캔버스에서 이벤트 설정 노드를 탐지 설정 노드(또는 AI 패키지·엣지 디바이스 노드)와 연결하고, 이벤트 발생 조건을 설정해 저장하는 절차를 설명한다.

## 절차

1. 탐지 설정 노드(또는 AI 패키지 노드, 엣지 디바이스 노드)의 출력 핸들을 이벤트 설정 노드의 입력 핸들에 연결한다. 연결 전에는 "이벤트를 설정할 탐지 설정 노드, 또는 AI 패키지를 연결해주세요" 안내만 표시된다.
2. 연결되면 헤더에 "개별 이벤트 설정" 또는 "융합 이벤트 설정"이 표시된다. 여러 데이터 소스를 하나의 이벤트로 묶은 경우 "융합 이벤트 설정"으로 표시된다.
3. 융합 이벤트 설정인 경우 "융합 이벤트 발생 기준 기간 설정" 드롭다운에서 선택 / 30초 / 1분 / 5분 / 10분 / 30분 / 60분 중 하나를 선택한다.
4. 연결된 리소스(센서, 데이터베이스, 탐지 설정 노드 등)의 측정 항목마다 "기준 설정" 값(1~1000)과 단위 시간(초/분/시), "발생 건수 (건)" 값(1~1000)을 입력한다.
5. 모든 항목을 입력한 뒤 "설정 저장" 버튼을 누른다.
6. 더 이상 연결이 필요 없으면 "전체 연결 해제" 버튼으로 연결된 엣지를 한 번에 모두 해제할 수 있다.

## 입력 시 주의 사항

- 기준 설정값과 발생 건수는 1 이상 1000 이하의 숫자만 입력할 수 있다. 범위를 벗어나면 입력란 아래 "1~1000까지 숫자만 입력 가능 합니다." 안내가 표시된다.
- "설정 저장"을 누를 때 다음 조건 중 하나라도 충족하지 않으면 토스트 오류가 표시되고 저장되지 않는다.
    - 융합 이벤트 설정에서 발생 기준 기간을 선택하지 않은 경우: "융합 이벤트 발생 기준 기간을 설정해주세요."
    - 기준 설정값을 입력하지 않은 항목이 있는 경우: "입력하지 않은 기준 설정값이 있습니다."
    - 발생 건수를 입력하지 않은 항목이 있는 경우: "입력하지 않은 발생 건수값이 있습니다."
    - 기준 설정값이 1~1000 범위를 벗어난 경우: "기준 설정값은 1~1,000 사이의 숫자만 입력 가능합니다."
    - 발생 건수가 1~1000 범위를 벗어난 경우: "발생 건수값은 1~1,000 사이의 숫자만 입력 가능합니다."

## 연결할 수 없는 경우

이벤트 설정 노드에 새 연결을 시도하면 다음 조건에 따라 연결이 막히고 안내 팝업이 표시된다.

- 연결하려는 노드의 데이터 연결 또는 설정이 완료되지 않은 경우: "해당 노드는 설정이 완료되지 않았습니다." / "해당 노드에 데이터 연결 또는 설정 완료 후 다시 연결을 시도해 주세요."
- 들어오는 연결이 탐지 설정 노드·AI 패키지 노드·엣지 디바이스 노드가 아닌 경우: "해당 노드를 연결할 수 없습니다."와 함께 연결 가능한 노드 종류를 안내한다.
- 나가는 연결이 알림 및 제어 노드(대시보드 푸시/모바일 푸시/SOP)가 아닌 경우에도 "해당 노드를 연결할 수 없습니다." 팝업이 표시된다.
- 같은 종류의 알림 및 제어 노드에 이미 연결되어 있는 상태에서 같은 종류를 또 연결하려는 경우: "해당 노드와 동일한 유형의 노드가 이미 연결되어 있습니다." / "다른 알림 및 제어 노드에 연결해주세요."

## 노드 공통 동작

이벤트 설정 노드를 포함한 모든 탐지 노드는 다음 공통 동작을 제공한다.

- 노드 이름 옆 연필 아이콘을 누르면 이름 변경 모드로 전환되며, 최대 20자까지 입력한 뒤 "저장" 버튼으로 변경한다.
- 최소화/최대화 아이콘으로 노드를 접거나 펼칠 수 있다.
- 닫기(x) 아이콘을 누르면 삭제 확인 팝업이 뜨고, 삭제를 확정하면 노드와 연결된 엣지가 함께 제거되며 "노드를 삭제했습니다." 안내가 표시된다. 모델 릴리즈 검증에 실패하면 "모델 릴리즈 검증에 실패했습니다." 안내와 함께 삭제가 취소된다.
- 노드 영역의 높이는 최대 708px이며, 그 이상은 스크롤로 표시된다.

## 정책-화면 점검 메모

화면설계서 정책 추출본(`event-setting-node.md`)에는 "탐지 영역명/데이터 소스명"(센서 AI 이상 탐지) 또는 "전체 영역/데이터 소스명"(영상 이상 탐지) 형태의 표시 규칙과, 열화상 카메라의 최고/최저/평균 온도 표시가 정의되어 있다. 이번 엔트리에서 화면 코드로 검증한 것은 센서 임계치 이벤트 항목(`SensorThresholdEvent.jsx`)뿐이며, 이 컴포넌트는 "탐지 설정" 배지 + 시나리오명, "센서"/"데이터베이스" 배지 + 데이터 소스명 형태로 표시한다. 영상 이상 탐지·열화상 카메라 등 다른 이벤트 유형의 표시 형식은 각각의 하위 컴포넌트(`VideoEvent.jsx`, `ThermographicThresholdEvent.jsx` 등)를 별도로 확인해야 하며, 이번 엔트리의 검증 범위에는 포함하지 않았다.
