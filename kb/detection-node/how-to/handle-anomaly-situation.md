# 이상 상황 알림 확인하기

헤더의 알림 센터에서 이상 상황 알림을 확인·정리하고, 실시간 이상 발생 알림에서 상세 화면으로 이동하는 절차를 설명한다. 알림은 이벤트 1건당 카드 1장으로 도착하며, 카드를 누르면 해당 화면으로 바로 이동할 수 있다.

## 절차

1. 화면 상단 헤더 우측의 **알림 개수 배지**를 누른다. 우측에 알림 센터가 열린다. 배지 숫자는 읽지 않은 알림 묶음 수이며, 100건 이상이면 "99+"로 표시된다.
2. **상황** 탭을 선택한다. 탐지 이벤트에서 만들어진 이상 상황 알림만 모아 볼 수 있다. 탭 우측 숫자가 읽지 않은 알림 수다.
3. 알림이 많으면 **상태**(전체/읽지 않음)·**유형** 필터로 추리고, **최신순** 버튼으로 정렬을 바꾼다(최신순, 오래된순, 심각도순 순환).
4. 알림 카드의 내용을 확인한다. 카드를 누르면 묶음 전체가 읽음 처리되고 상황 상세 페이지로 이동한다.
5. 확인을 마친 알림은 **모두 읽음**으로 일괄 정리한다. 화면에 필터링된 읽지 않은 알림이 한 번에 읽음 처리된다.
6. 실시간으로 도착하는 알림은 헤더의 **종 아이콘**으로 여는 **이상 발생 알림** 드로어에서 확인한다. 항목의 **상세 보기**를 누르면 해당 화면으로 이동한다.

## 알림 유형

상황 탭에는 6가지 상황 이벤트 알림(신규 상황, 심각도 상승, 상황 업데이트, 처리 지연, 이의 제기, 담당자 배정)이 표시된다. 카드의 안내문이 해야 할 일을 알려준다. 자주 받는 3가지 유형의 안내문은 다음과 같다.

| 유형 | 카드 안내문 | 해야 할 일 |
|---|---|---|
| 신규 상황 | "현장을 확인하고 담당자를 배정해 주세요." | 현장 확인 후 담당자 배정 |
| 심각도 상승 | "상황의 위험도가 높아졌습니다. 즉시 현장을 확인하고 조치를 진행해 주세요." | 즉시 확인·조치 |
| 상황 업데이트 | "상황이 업데이트되었습니다. 변경된 내용을 확인해 주세요." | 변경 내용 확인 |

처리 지연·이의 제기·담당자 배정 카드의 안내문은 상황별로 에이전트가 생성한다.

시스템 탭에는 모델 배포 실패, 데이터 수집 중단, 탐지 시나리오 강제 종료 알림이 표시된다.

## 알림을 누르면 이동하는 화면

| 알림 유형 | 이동 화면 |
|---|---|
| 상황 이벤트 알림 (신규 상황·심각도 상승 등 6종) | 상황 상세 페이지 |
| 데이터 수집 중단 | 연결 데이터 관리 |
| 탐지 시나리오 강제 종료 | 탐지 시나리오 관리 |
| 모델 배포 실패 | 모델 관리 |

## 확인 전 알아두기

- 알림 카드는 발송 시점 내용으로 고정된다. 카드를 받은 뒤 상황 제목이 바뀌어도 카드는 갱신되지 않으므로, 최신 내용은 카드를 눌러 상세 화면에서 확인한다.
- 읽음 상태는 개인별로 저장되고, 데스크톱과 모바일 간에 묶음 단위로 즉시 동기화된다.
- "총 N건"은 최근 30일 기간의 알림 수다.
- 알림 시각은 경과에 따라 "방금 전", "n분 전", "n시간 전", "n일 전"으로 표시되고, 7일이 지나면 날짜(MM-DD)로 표시된다.

## 개편 예정 미리보기

이상 상황을 문제 단위로 묶어 확인·조치·감사까지 한 흐름으로 처리하는 "이상 상황 레이어" 개편이 프로토타입으로 공개되어 있다. 개편 내용과 이 가이드가 개편을 추적하는 방식은 [이 가이드가 만들어지는 방식](../explanation/anomaly-alert-guide-sourcing.html)에서 설명한다.
