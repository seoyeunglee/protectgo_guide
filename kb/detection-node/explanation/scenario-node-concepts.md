# 탐지 시나리오와 노드 — 개념 이해

Protect Go의 탐지는 코드나 설정 파일이 아니라 **노드를 연결한 그림**으로 정의한다. 이 문서는 노드가 무엇이고, 데이터가 어떻게 알림이 되는지 개념을 설명한다. 실제 만드는 절차는 [첫 탐지 시나리오 만들기](../tutorial/create-first-scenario.html)에서 다룬다.

## 탐지 시나리오란

탐지 시나리오는 "어떤 데이터를, 어떤 기준으로 감시하고, 이상이 생기면 누구에게 알릴지"를 하나의 흐름으로 묶은 단위다. 시나리오 하나가 캔버스 한 장이고, 캔버스 위의 노드들이 그 흐름을 구성한다.

## 데이터가 알림이 되기까지

노드는 항상 왼쪽에서 오른쪽으로 흐른다. 입력(데이터·엣지)에서 시작해 판정(탐지 설정)과 조건(이벤트 설정)을 거쳐 출력(알림 및 제어)으로 끝난다.

1. **데이터 노드**가 센서·카메라의 측정값과 영상을 받아들인다.
2. **탐지 설정 노드**(또는 AI 패키지 노드)가 그 데이터를 기준과 비교해 이상 여부를 판정한다.
3. **이벤트 설정 노드**가 "기준값을 몇 번 넘으면 이벤트로 볼 것인지" 발생 조건을 정의한다.
4. **알림 및 제어 노드**가 발생한 이벤트를 대시보드 푸시, 모바일 푸시, SOP 등으로 전달한다.

## 노드 6종의 역할

| 노드 | 역할 | 흐름에서의 위치 |
|---|---|---|
| 엣지 | 엣지 디바이스 단위 탐지 | 입력 |
| 데이터 | 센서·카메라 입력 연결 | 입력 |
| AI 패키지 | 모델+데이터+설정을 묶은 탐지 단위 | 판정 |
| 탐지 설정 | 영상/임계치/통계 탐지 기준 정의 | 판정 |
| 이벤트 설정 | 이벤트 발생 조건(기준 설정값·발생 건수) 정의 | 조건 |
| 알림 및 제어 | 알림·제어 액션 연결 | 출력 |

영상 탐지에는 관심 영역을 지정하는 **ROI 노드**가 추가로 쓰인다.

## 탐지 시나리오 5종

감시하려는 데이터에 따라 시나리오 유형이 정해진다.

| 유형 | 데이터 소스 | 판정 방식 |
|---|---|---|
| 영상 이상 탐지 | CCTV·열화상 영상 | 비전 AI 모델 (작업자 안전 모델 등) |
| DB 임계치 탐지 | 외부 DB | 수치 임계치 비교 |
| 센서 임계치 탐지 | 진동·온도·복합가스·열화상 센서 | 수치 임계치 비교 |
| 통계 분석 탐지 | 센서(다중) | Z-Score / 변화율 / EWMA 통계 이상치 |
| 발전 효율 저하 탐지 | 전력 센서 + 발전량 데이터 | 발전량 예측값 대비 실측값 비교 |

## 왜 노드 방식인가

- **흐름이 보인다**: 데이터부터 알림까지 한 화면에서 추적할 수 있어, 알림이 왜 왔는지(또는 왜 안 왔는지) 연결을 따라가며 확인할 수 있다.
- **조합이 자유롭다**: 같은 데이터 노드를 여러 탐지 설정에 연결하거나, 여러 데이터를 하나의 이벤트로 묶는(융합 이벤트) 구성을 그림으로 만들 수 있다.
- **검증이 내장된다**: 연결할 수 없는 노드 조합은 연결 시점에 차단되고 안내 팝업이 표시된다 — 자세한 규칙은 [이벤트 설정 노드 구성하기](../how-to/configure-event-setting-node.html)의 "연결할 수 없는 경우"를 참고한다.
