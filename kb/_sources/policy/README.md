# 정책 텍스트 추출분 (`kb/_sources/policy/`)

화면설계서 PDF에서 **정책 신호가 있는 섹션만** 추려 담는 곳이다 (CLAUDE.md "화면설계서 PDF 읽기 규칙").
KB 엔트리의 `policy` 필드는 PDF 원본이 아니라 이 폴더의 파일만 거쳐서 채운다.

## 생성 절차 (요약 — 2단계 본 작업에서 수행)
1. `extracted/<pdf명>/page-NN.txt` — 작은 스크립트(pymupdf/pdftotext)로 텍스트만 추출, 페이지 단위 저장 (캐시, `.gitignore` 대상)
2. 추출 텍스트에서 정책 신호 키워드(상태/전환/조건/예외/기본값/유효성/권한/제약/오류/필수 등)가 있는 섹션만 grep/정규식으로 1차 선별
3. 선별된 정책 텍스트를 화면 단위로 정리해 이 폴더에 `<screen>.md`로 저장

## 파일명 규칙
원본 PDF(=화면) 이름 기준. 예: `프로젝트 생성 -_ 1. 프로젝트 이름 입력.pdf` → `project-creation-1-name.md`
(한글 파일명 대신 도메인에서 쓰는 슬러그를 사용해 KB·`provenance.ref`에서 참조하기 쉽게 한다.)

## 파일 형식
```yaml
---
screen: 프로젝트 생성 - 1. 프로젝트 이름 입력
source_pdf: "Figma_PDF_files/프로젝트 생성 -_ 1. 프로젝트 이름 입력.pdf"
pages: [1, 2]
extracted_at: "2026-06-08"
---

(정책 신호가 있는 섹션만 — 상태 전환·수식·예외·기본값·유효성·권한·제약·오류·필수 등)
```

이번 도메인(프로젝트 생성·탐지 시나리오·노드)에 해당하는 화면만 추린다. 다른 화면은 다루지 않는다.
