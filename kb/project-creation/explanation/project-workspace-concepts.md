# 프로젝트와 워크스페이스 — 개념 이해

Protect Go에서 가장 먼저 만나는 단위가 프로젝트다. 이 문서는 프로젝트가 무엇이고, 소유·참여·권한·생명주기가 어떻게 연결되는지 설명한다. 만드는 절차는 [첫 프로젝트 만들기](../tutorial/create-first-project.html)에서 다룬다.

## 프로젝트는 격리된 작업 공간이다

프로젝트는 한 고객사·현장 단위의 워크스페이스다. 데이터베이스, 탐지 설정, 멤버가 모두 프로젝트별로 분리되어, 한 프로젝트의 센서 데이터나 시나리오가 다른 프로젝트에 보이지 않는다. 여러 현장을 관리한다면 현장마다 프로젝트를 따로 만든다.

## 소유와 참여는 다르다

| 구분 | 정의 | 한도 |
|---|---|---|
| 소유 | 프로젝트를 생성했거나 어드민 초대로 받은 최초 에디터 | 계정당 최대 3개 |
| 참여 | 멤버 초대로 합류한 사용자 | 제한 없음 |

소유 한도 3개에 도달하면 새 프로젝트 생성 버튼이 비활성화된다. 프로젝트 선택 페이지도 "내 관리 프로젝트"와 "참여 중인 프로젝트"를 구분해 보여준다.

## 권한은 프로젝트 안에서 정해진다

같은 사람이라도 프로젝트마다 다른 권한(에디터·매니저·뷰어)을 가질 수 있다. 권한은 그 프로젝트 안에서 무엇을 할 수 있는지를 정한다 — 자세한 범위는 [권한별 기능 레퍼런스](../reference/permission-matrix.html)에서 확인한다. 예를 들어 매니저는 자신이 담당한 장소의 상황만 열람한다.

## 생명주기 — 생성에서 비활성화까지

1. **생성**: 이름 입력, 멤버 초대, 사용 용도 선택의 3단계 위저드를 거친다.
2. **활성**: 멤버가 접속해 탐지 시나리오를 구성하고 운영한다.
3. **비활성화**: 어드민이 임시로 차단할 수 있다. 비활성화된 프로젝트는 카드에 "서비스 이용 일시 중지"가 표시되고 접속할 수 없다. 헤더의 프로젝트 목록에서도 "일시 중지" 배지가 붙는다 — [다른 프로젝트로 전환하기](../how-to/switch-project.html)의 비활성화 절을 참고한다.

로컬 서버로 생성하는 경우에는 생성이 끝날 때까지 카드에 진행률("데이터 베이스를 구성중 입니다." + 퍼센트)이 표시되고, 실패하면 [생성 재시도] 버튼이 제공된다.

## 왜 프로젝트 단위인가

- **보안·격리**: 현장별 데이터가 섞이지 않는다. 멤버는 초대받은 프로젝트만 본다.
- **권한의 경계**: 권한이 프로젝트 단위로 부여되어, 같은 계정으로 현장마다 다른 역할을 수행할 수 있다.
- **운영의 경계**: 비활성화·라이선스 관리가 프로젝트 단위로 이루어져, 한 현장의 중지가 다른 현장에 영향을 주지 않는다.
