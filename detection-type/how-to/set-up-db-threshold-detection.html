DB 임계치 탐지 설정하기

외부 DB(InfluxDB)에 저장된 센서 측정값을 데이터베이스 노드로 불러와 임계치로 모니터링하는 방법

DB 임계치 탐지는 센서를 직접 연결하는 대신, 외부 데이터베이스(InfluxDB)에 이미 쌓이고 있는 측정값을 가져와 임계치로 감시하는 방식이다. 별도 수집 장비 없이 기존 데이터 인프라를 그대로 활용할 때 쓴다.

설정 절차

  1. 시나리오에 데이터베이스 노드를 놓고, 모니터링할 외부 DB의 측정값을 연결한다.
  2. 임계치 설정 노드를 연결한다.
  3. 임계치를 입력한다 — 숫자(점 1개 포함, 소수점 셋째 자리까지)로 측정 범위 안의 값을 넣고, "이상" 또는 "이하" 기준을 정한다.
  4. 이벤트 설정 노드로 발생 조건을 정하고 알림 노드를 붙인다.

임계치 입력 규칙은 센서 임계치 탐지와 동일하다. 측정 범위를 벗어나거나 빈 값으로 저장하면 에러가 표시된다.

센서 임계치 탐지와 무엇이 다른가

판정 방식(값이 선을 넘으면 탐지)은 같지만, 데이터를 어디서 가져오는가가 다르다.

  • 센서 임계치 탐지: Protect Go에 등록된 센서에서 실시간으로 값을 받는다.
  • DB 임계치 탐지: 외부 DB(InfluxDB)에 저장된 값을 읽어 온다.

기존에 운영 중인 센서 데이터 파이프라인이 있다면 DB 임계치 탐지로 연동하는 편이 빠르다.

다음 단계

  • DB 연결 자체를 등록·관리하려면 연결 데이터 관리(후속 도메인)를 참고한다.
  • 직접 연결된 센서를 임계치로 보려면 센서 임계치 탐지 설정하기를 본다.
