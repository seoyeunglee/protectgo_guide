통계 분석 탐지 설정하기 (Z-Score·변화율·EWMA)

고정 임계치 대신 평소 패턴 대비 이상치를 잡는 통계 분석 탐지(Z-Score·변화율·EWMA)를 설정하고 탐지 데이터를 읽는 방법

통계 분석 탐지는 "고정된 선"이 아니라 평소 패턴에서 얼마나 벗어났는가로 이상을 잡는다. 절대값은 정상 범위라도 갑자기 튀거나 추세가 어긋나면 탐지한다. 임계치만으로는 잡기 어려운 미묘한 이상에 쓴다.

세 가지 분석 방식

유형 무엇을 보나
Z-Score 탐지 평균 대비 표준편차의 몇 배만큼 벗어났는가
변화율 탐지 이전 시점 대비 얼마나 급격히 변했는가
EWMA 탐지 지수가중이동평균으로 누적 오차의 추세가 어긋나는가

설정 절차

  1. 시나리오에 분석할 센서 데이터(또는 해당 AI 패키지)를 연결한다.
  2. 통계 분석 탐지 노드를 구성하고, 위 세 방식 중 측정 항목에 맞는 것을 적용한다.
  3. 이벤트 설정 노드로 발생 조건을 정하고 알림 노드를 붙인다.

탐지 데이터 읽는 법

통계 탐지는 그래프로 읽는 게 핵심이다.

  • 센서 태그로 측정 항목을 골라 본다.
  • 평균값 점선: 365일 기준 평균값이 점선으로 깔린다 — 이게 "평소"의 기준선이다.
  • 붉은 점: 이번 탐지 데이터 기준점이다. 평균선에서 얼마나 떨어졌는지로 이상 정도를 가늠한다.
  • 동일 유형 탐지 이력: 같은 장소·센서·항목의 과거 이상 탐지가 점으로 함께 표시되어, 단발성인지 반복되는 문제인지 맥락을 본다.
  • 센서값 박스: "센서값/센서평균" 형식(예: "654°C /432°C")으로 현재값과 평균을 나란히 보여준다.

그래프 X축은 이벤트 발생 시점 기준 4시간 전 ~ 2시간 후를 보여주고, 점에 마우스를 올리면 데이터 소스명·기준점 기간(10분 단위)·발생 시간·센서값이 툴팁으로 뜬다.
