튜토리얼

첫 프로젝트 만들기

로그인 후 프로젝트 생성 3단계(이름 입력, 멤버 초대, 사용 용도 선택)를 거쳐 첫 프로젝트를 만들고 대시보드에 진입하는 과정

Protect Go의 모든 작업은 프로젝트 안에서 이루어진다. 프로젝트는 한 고객사·현장 단위의 작업 공간이며, 데이터·설정·멤버가 프로젝트별로 분리된다. 이 튜토리얼은 로그인부터 첫 프로젝트 생성, 대시보드 진입까지를 다룬다.

프로젝트 선택·생성 화면은 별도 인증 애플리케이션에서 제공되어 이 가이드에는 화면 캡처가 없다. 화면 명칭과 동작은 정책 문서 기준으로 안내한다.

절차

  1. 로그인한다. 소속된 프로젝트가 2개 이상이면 프로젝트 선택 페이지로 진입하고, 처음 사용자는 프로젝트 생성으로 안내된다.
  2. 새 프로젝트 생성을 누른다. 한 계정이 소유할 수 있는 프로젝트는 최대 3개이며, 3개를 소유 중이면 버튼이 비활성화된다.
  3. 1단계에서 프로젝트명을 입력한다. 2자 이상 20자 미만으로 입력하면 다음 버튼이 활성화된다. 내가 소유한 다른 프로젝트와 이름이 같으면 "이미 사용 중인 프로젝트 이름입니다." 안내가 표시된다.
  4. 2단계에서 함께 일할 멤버를 초대한다. 이메일을 입력하고 ENTER 키 또는 추가 버튼으로 목록에 담는다. 한 번에 최대 10명까지 추가할 수 있고, 이 단계에서 초대한 멤버의 권한은 모두 매니저로 지정된다. 나중에 초대하려면 이 단계 건너뛰기를 누른다.
  5. 3단계에서 사용 용도를 선택한다. 건설 현장, 제조 현장, 기타(직접 입력, 최대 50자) 중 하나를 고른다.
  6. 생성을 완료하면 진행률 팝업(0%, 70%, 90%, 100%)을 거쳐 새 프로젝트의 대시보드로 진입한다.

시작 전 알아두기

  • 프로젝트 "소유"와 "참여"는 다르다. 소유는 최대 3개로 제한되지만, 멤버로 참여하는 프로젝트 수는 제한이 없다.
  • 생성 중 멤버 초대를 건너뛰어도, 생성 후 계정 관리 화면에서 언제든 초대할 수 있다. 이때는 에디터/매니저/뷰어 중 권한을 선택할 수 있다 — 멤버 초대하기를 참고한다.
  • 접속 진행률 팝업이 15초 이상 멈춰 있으면 접근 실패 안내가 표시된다. 사내 네트워크(내부 IP) 또는 VPN 연결 상태를 확인한다.

다음 단계

  • 멤버별 권한이 어떤 기능을 쓸 수 있는지는 권한별 기능 레퍼런스에서 확인한다.
  • 프로젝트가 준비됐다면 첫 탐지 시나리오 만들기로 이어진다.
