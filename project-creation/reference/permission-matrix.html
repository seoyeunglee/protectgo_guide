권한별 기능 레퍼런스

어드민·에디터·매니저·뷰어 4가지 권한의 정의와 기능별 사용 가능 범위

Protect Go의 권한은 4종이다. 멤버를 초대하거나 권한을 변경하기 전에, 각 권한이 어디까지 할 수 있는지 이 표에서 확인한다.

권한 4종

권한 대상 사이드바 접근 범위 핵심 권한
어드민 공급사(IDB) 내부 운영자 별도 어드민 페이지 프로젝트 생성, 디바이스 관리, 라이선스 관리
에디터 고객사 관리자 전체 메뉴 모든 페이지·기능 접근, 멤버 초대, 위젯 편집, 시나리오 편집
매니저 현장 담당자·실무자 대시보드 + 통계 담당 장소 설정, 안전 체크 처리
뷰어 일반 조회자 대시보드만 모든 기능 조회만 (담당 장소 비활성)

기능별 사용 가능 범위

기능 어드민 에디터 매니저 뷰어
프로젝트 생성·삭제 가능 가능 (소유자) 불가 불가
멤버 초대 가능 가능 불가 불가
권한·소속·담당 장소 변경 가능 가능 불가 불가
탐지 시나리오 편집 가능 가능 불가 불가
모델 배포 관리 가능 가능 불가 불가
대시보드 위젯 편집 해당 없음 가능 불가 불가

안전 체크리스트 "확인 시작" 해당 없음 미확정* 가능 (담당 장소만) 불가
통계 페이지 조회 가능 가능 가능 불가

*미확정: 사양서에 명시되지 않은 항목 — 실제 운영 시 확인이 필요하다.

권한 변경 시 동작

  • 권한 변경은 계정 정보 수정 팝업 또는 일괄 변경 팝업에서 진행한다.
  • 에디터로 변경하면 담당 장소가 "전체" 1개로 자동 지정되고, 개별 장소는 지정할 수 없다.
  • 매니저로 지정할 때만 담당 장소 선택 셀렉박스가 활성화된다.
  • 뷰어를 선택하면 소속·담당 장소 셀렉박스가 비활성화된다.

관련 가이드

  • 멤버를 초대하는 절차는 멤버 초대하기를 참고한다. 프로젝트 생성 중 초대는 권한이 매니저로 고정되고, 계정 관리 경로의 초대는 에디터/매니저/뷰어 중 선택할 수 있다.
