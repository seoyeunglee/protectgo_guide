프로젝트 생성·관리

Protect Go에서 가장 먼저 만나는 단위가 프로젝트다. 프로젝트는 한 고객사·현장 단위의 워크스페이스이며, 데이터베이스·탐지 설정·멤버가 모두 프로젝트별로 격리된다 — 한 프로젝트의 센서 데이터나 시나리오는 다른 프로젝트에 보이지 않는다. 여러 현장을 관리한다면 현장마다 프로젝트를 따로 만든다.

소유와 참여는 다르다

구분 정의 한도
소유 프로젝트를 생성했거나 어드민 초대로 받은 최초 에디터 계정당 최대 3개
참여 멤버 초대로 합류한 사용자 제한 없음

소유 한도 3개에 도달하면 새 프로젝트 생성 버튼이 비활성화된다.

권한은 프로젝트 안에서 정해진다

같은 사람이라도 프로젝트마다 다른 권한(에디터·매니저·뷰어)을 가질 수 있다. 권한은 그 프로젝트 안에서 무엇을 할 수 있는지를 정한다. 예를 들어 매니저는 자신이 담당한 장소의 상황만 열람한다.

생명주기 — 생성에서 비활성화까지

  1. 생성: 이름 입력, 멤버 초대, 사용 용도 선택의 3단계 위저드를 거친다.
  2. 활성: 멤버가 접속해 탐지 시나리오를 구성하고 운영한다.
  3. 비활성화: 어드민이 임시로 차단할 수 있다. 비활성화된 프로젝트는 카드에 "서비스 이용 일시 중지"가 표시되고 접속할 수 없다.

로컬 서버로 생성하는 경우에는 생성이 끝날 때까지 카드에 진행률("데이터 베이스를 구성중 입니다." + 퍼센트)이 표시되고, 실패하면 [생성 재시도] 버튼이 제공된다.

튜토리얼

  • 첫 프로젝트 만들기 — 로그인 후 프로젝트 생성 3단계(이름 입력, 멤버 초대, 사용 용도 선택)를 거쳐 첫 프로젝트를 만들고 대시보드에 진입하는 과정

하우투

  • 다른 프로젝트로 전환하기 — 헤더의 프로젝트 선택 드롭다운에서 소속된 다른 프로젝트로 전환하는 방법
  • 멤버 초대하기 — 계정 관리 화면에서 새 멤버에게 프로젝트 참여를 요청하는 방법
  • 장소·구역·설비 관리하기 — 프로젝트 설정의 장소 관리에서 구역과 장소를 등록하고, 장소에 설비를 추가하고, 이름 변경·삭제하는 방법
  • 프로젝트 설정 변경하기 (이름·초대 코드·데이터 저장) — 프로젝트 설정 페이지에서 프로젝트 이름을 변경하고, 초대 코드를 확인·복사하고, 데이터 저장 경로·자동 삭제·주간 리포트를 관리하는 방법
